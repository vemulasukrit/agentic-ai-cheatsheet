① Orchestration Layer
CRITICAL
Why: Agents must never call each other directly. All coordination flows through a central orchestrator — prevents coupling & enables retries.
Required Files
src/orchestration/ workflow_engine.py # DAG-based task orchestration task_scheduler.py # Priority queues + retries agent_router.py # Routes tasks → correct agent supervisor_agent.py # Hierarchical control pipeline.py # Sequential / parallel runs

② Tool / Action Registry
CRITICAL
Why: Every tool must be typed, validated, and discoverable. Ad-hoc function calls break at scale and are impossible to test in isolation.
Required Files
src/tools/ tool_registry.py # Central catalogue + metadata base_tool.py # Abstract tool interface tool_validator.py # Input/output JSON schema built_in/ web_search.py code_executor.py file_handler.py api_caller.py
Key Rule
Every tool must expose: name · description · input_schema · output_schema · execute()

③ Safety & Guardrails
CRITICAL
Why: Required by EU AI Act & NIST AI RMF. No agentic system should reach production without input/output/action guards.
Required Files
src/safety/ input_guardrails.py # Sanitize agent inputs output_guardrails.py # Filter harmful outputs rate_limiter.py # Stop runaway loops action_validator.py # Approve/reject destructive ops human_in_the_loop.py # Pause for human approval content_policy.py # Domain-specific rules

EU AI Act NIST AI RMF ISO 42001

④ Stratified Memory
IMPORTANT
Why: A flat data/memory/ folder fails for long-running agents. Memory must be stratified into 4 distinct tiers.
4-Tier Memory Model
src/memory/ working_memory.py # Short-term, in-context state episodic_memory.py # Past interaction episodes semantic_memory.py # Facts + vector DB embeddings procedural_memory.py # Learned behaviors/skills memory_consolidator.py # working → long-term pipeline retrievers/ vector_retriever.py bm25_retriever.py hybrid_retriever.py
⚡
⑤ LLM Provider Gateway
IMPORTANT
Why: model_config.yaml alone means swapping models requires touching every agent. You need a provider-agnostic gateway.
Required Files
src/llm/ base_llm.py # Abstract interface llm_router.py # Cost / latency routing prompt_manager.py # Versioned prompt templates token_counter.py # Budget management cache.py # Semantic response caching providers/ openai_provider.py anthropic_provider.py ollama_provider.py # Local models
📡
⑥ Agent Communication
IMPORTANT
Why: Without a message bus, collaborative agents become tightly coupled. Direct calls cannot be traced, retried, or scaled.
Required Files
src/communication/ message_bus.py # Pub/sub between agents message_schema.py # Standardised message format protocols/ request_reply.py broadcast.py consensus.py # Multi-agent agreement
Message Schema Must Include
id · sender · recipient · type · payload · timestamp · correlation_id · ttl

⑧ State Management
RECOMMENDED
Why: Without a proper FSM, long-running agents cannot recover from failures mid-execution. Every agent needs a lifecycle.
Required Files

Production Best Practices — 20 Golden Rules
Architecture
  • ①
    Never direct-call between agents
    All agent interaction goes through the orchestrator or message bus
  • ②
    Every tool must have a schema
    name, description, input_schema, output_schema, execute()
  • ③
    Abstract your LLM provider
    base_llm.py interface — swap models without touching agents
  • ④
    Stratify memory into 4 tiers
    working → episodic → semantic → procedural
  • ⑤
    Use YAML configs for everything
    agent_config, model_config, environment_config, logging_config
  • ⑥
    Implement FSM for agent lifecycle
    idle → running → waiting → failed → done
  • ⑦
    Checkpoint long-running agents
    Save state periodically to enable mid-execution recovery
Safety
  • ⑧
    Guard every input AND output
    Sanitise before LLM call; filter after — never skip either
  • ⑨
    Rate-limit recursive agent calls
    Max depth + max iterations enforced at orchestrator level
  • ⑩
    Human-in-the-loop for destructive ops
    Delete, write, send, pay — always require explicit approval
Testing
  • ⑪
    Always mock the LLM in unit tests
    mock_llm.py returns deterministic fixtures — no API calls in CI
  • ⑫
    Test each layer independently
    Unit → Integration → E2E — never skip integration tests
  • ⑬
    Benchmark reasoning & latency
    Track performance regressions per agent type across releases
Observability
  • ⑭
    Trace every agent execution
    OpenTelemetry spans from trigger → tools used → final output
  • ⑮
    Log immutable audit trails
    Every decision the agent makes must be permanently recorded
  • ⑯
    Track token cost per run
    cost_tracker.py — prevent runaway bills in production
Deployment
  • ⑰
    Expose agents via REST + WebSocket
    Sync for simple tasks; streaming WS for long-running responses
  • ⑱
    Use semantic prompt caching
    cache.py — deduplicate similar queries to reduce cost by 40–60%
  • ⑲
    Version control your prompts
    prompt_manager.py with git-tracked templates — treat like code
  • ⑳
    Automate CI/CD for agent pipelines
    test.yml triggers on every PR; deploy.yml gates on benchmark pass
